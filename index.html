Gist

I have used Gist for development of code to test various random bits and pieces. So far there is some Python code snippets and some RFO Basic! (Basic interpretter on Android phone) snippets on Gist. Since I am the only person working on the code I am only using the Master branch. The workflow for the single file of python code on my home PC is very simple -
edit & test code, git add code, git commit, git push and then repeat.
Some of the time I edit in the web browser, then do a git pull to test the code.

Github

I have also used Github for development of more substantial programs. For example, I have worked on a command line file dump utility, named ihad using Github. The ihad program dumps the bytes in a file as lines of "index", "hexadecimal" and "ASCII" formatted text. It gave me an opportunity to try creating a branch and then merging it back into the master branch. Didn't work as expected, so more reading required. It also was a trial of using RCS and git in the same project. Didn't help and only led to confusion. I also have put an Excel workbook on it that helps a person solve Sudoku puzzles and a python script named checktime.py that attempts to check the time on another networked device by sending an (IPv4) ICMP timestamp request and waiting for a corresponding timestamp reply. There is also a pingDemo in the go language, which attempts to determine if another network device is alive by sending an (IPv4) ICMP echo request (i.e. ping) and waiting for a corresponding ICMP echo reply.

Volunteer Computing using BOINC

Some of my computing devices are available for use to Scientific computing bodies such as World Community Grid, Einstein@Home, Asteroids@Home. Experimenting with BOINC settings on various devices resulted in me developing a preference to setting aside a proportion of the available CPUs, but making those CPUs 100% available.
